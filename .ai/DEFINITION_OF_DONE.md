# Definition of Done

- [ ] História e regra de negócio atendidas sem escopo lateral.
- [ ] Caso de uso e domínio não dependem de framework.
- [ ] Testes unitários cobrem invariantes, limites e transições inválidas.
- [ ] Teste de integração usa PostgreSQL/Testcontainers.
- [ ] Autorização e isolamento multi-tenant possuem testes negativos.
- [ ] Migration, índices e constraints revisados.
- [ ] OpenAPI, exemplos e Problem Details RFC 9457 atualizados.
- [ ] Auditoria, logs e métricas não expõem dados sensíveis.
- [ ] UI possui estados de carregamento, vazio, erro, sucesso e acessibilidade.
- [ ] Fluxo principal E2E aprovado.
- [ ] Sem segredo, TODO sem issue, código morto ou supressão injustificada.
- [ ] Documentação e ADR atualizados quando houve decisão.

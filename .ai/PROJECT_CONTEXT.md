# Contexto do projeto

O produto administra o ciclo cervejeiro completo: cadastros, água, receitas, planejamento, brassagem assistida, sanitização, fermentação, envase, estoque, qualidade, rastreabilidade, custos e IA.

O primeiro usuário é um cervejeiro/desenvolvedor operando em pequena escala, com perfil de equipamento como a SB40/50 L. O projeto deve continuar simples para uma pessoa desenvolver, mas sem comprometer rastreabilidade, testes ou limites de módulo. A arquitetura deve pagar pelo problema que resolve: abstração sem risco, variação ou regra real é desperdício.

## Resultado esperado do MVP

Um usuário consegue cadastrar equipamentos e insumos, publicar uma receita, liberar uma ordem, reservar estoque, executar a brassagem, registrar fermentação, sanitizar equipamentos, envasar, consumir lotes e encerrar o lote com custo e rastreabilidade.

## Princípio do produto

O plano é uma intenção; o executado é evidência. Nunca sobrescrever o planejado com o realizado. Toda recomendação deve mostrar entradas, método, impacto e nível de confiança.

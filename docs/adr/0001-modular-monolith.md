# ADR — Monólito modular

Status: Aceito

## Contexto

O sistema precisa crescer sem impor complexidade operacional incompatível com um desenvolvedor solo.

## Decisão

Adotar um deploy único com módulos fortemente delimitados.

## Motivo

Reduz operação e coordenação para desenvolvimento solo, preservando extração futura.

## Consequências

A decisão é obrigatória até que um novo ADR apresente evidência, migração e impacto de reversão.
